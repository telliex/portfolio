# i18n + Theme/Language Controls Design

**Date:** 2026-05-09
**Status:** Approved

## Summary

Replace the current "show both languages simultaneously" approach with a proper EN/ZH language switcher. Merge the existing 3-button theme toggle into a single dropdown. Move both controls (theme + language) from the footer into the header top-right. Footer becomes text-only.

---

## Goals

- One language visible at a time (EN or ZH), user-selectable
- Theme control: single dropdown (Light / Dark / System) replacing the current 3-icon animated row
- Language control: dropdown (EN / 中文) next to the theme dropdown
- Both controls in the header (top-right), not the footer
- Language and theme preferences persist via `localStorage`
- Zero new npm dependencies

---

## Architecture

### New Files

**`lib/i18n.ts`**
Typed translation object `Record<'en' | 'zh', Translations>`. Contains all static strings from `app/page.tsx` that currently appear duplicated in both English and Chinese:
- Intro paragraphs (3 × EN, 3 × ZH)
- Services section headings and descriptions (4 services × EN+ZH)
- Partners section heading and body paragraphs
- Section headings: "Selected Projects", "Work Experience", "Blog", "Connect", "Contact Form"
- Header subtitle: "Innovative Technology, Exceptional Solutions" / "創新技術，卓越解決方案"
- Footer: "Built with Motion-Primitives." (same in both, no translation needed)
- Connect section intro text and email label

**`lib/language-context.tsx`**
- `LanguageProvider` component — reads `localStorage` key `"lang"` on mount (defaults to `"en"`), stores in React state, writes back on change
- `useLanguage()` hook — returns `{ lang: 'en' | 'zh', setLang }`
- `useTranslations()` hook — returns the full translations object for the active language

### Modified Files

**`app/layout.tsx`**
- Wrap `<ThemeProvider>` children with `<LanguageProvider>`

**`app/header.tsx`**
- Add two dropdowns to the right side of the header row
- `ThemeDropdown`: single button showing current theme icon (☀️ / 🌙 / 🖥) + chevron; opens a dropdown with Light / Dark / System options; active item has subtle background fill + gray text
- `LangDropdown`: button showing `EN` or `中` + chevron; opens a dropdown with EN / 中文; same active-item style (gray text, subtle bg — no bold)
- Both dropdowns close on outside click (`useClickOutside` hook already exists at `hooks/useClickOutside.tsx`)
- Dropdowns styled with zinc palette to match existing site: `border-zinc-200`, `bg-white`, `dark:bg-zinc-900`, `dark:border-zinc-800`, shadow

**`app/footer.tsx`**
- Remove `ThemeSwitch` component entirely
- Footer becomes: left = `TextLoop` copyright, right = nothing (or removed right side)
- Clean, text-only footer

**`app/page.tsx`**
- Import `useLanguage` and `useTranslations`
- Remove all hardcoded ZH duplicate blocks (the `<div className="flex-1">` blocks with Chinese paragraphs)
- Replace static EN strings with `t.intro`, `t.services`, etc. from `useTranslations()`
- Project content: use `project.description[lang === 'en' ? 0 : 1]` and `project.list[lang]` (no change to `data.ts` types needed)

**`app/header.tsx`** subtitle
- Replace the hardcoded `"Innovative Technology, Exceptional Solutions"` string with `t.headerSubtitle` from `useTranslations()`

## UI Detail

### Header layout
```
[ Activo | Telliex Chiu          ]  [ ☀️ ▾ ] [ EN ▾ ]
[ Innovative Technology... subtitle ]
```

### Dropdown behaviour
- Opens on button click, closes on outside click or selecting an item
- Only one dropdown open at a time (opening one closes the other)
- Position: `absolute`, right-aligned, `top-full mt-1`, `z-50`
- Width: theme ~120px, language ~90px

### Active item style (both dropdowns)
- Background: `bg-zinc-100 dark:bg-zinc-800`
- Text: `text-zinc-500 dark:text-zinc-400` (gray, not bold)
- Inactive items: same gray text, no background, hover gets `hover:bg-zinc-50 dark:hover:bg-zinc-800/50`

## Data Flow

```
localStorage("lang")
      │
LanguageProvider  ←→  setLang()  ←  LangDropdown (header)
      │
useLanguage() → lang
      │
useTranslations() → t  →  page.tsx (static text)
                       →  header.tsx (subtitle)
                       →  project.description[lang idx]
                       →  project.list[lang]

localStorage("theme")
      │
next-themes ThemeProvider  ←→  setTheme()  ←  ThemeDropdown (header)
```

---

## Out of Scope

- URL-based locale routing (`/en/`, `/zh/`)
- Adding more than 2 languages
- Translating project `name` field or work experience titles (English-only, acceptable)
- Blog post translation
